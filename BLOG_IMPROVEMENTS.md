# 블로그 개선점 조사 보고서

## 개요
Jekyll 기반 GitHub Pages 블로그의 전반적인 개선점을 조사한 결과입니다.

---

## 1. SEO (검색 엔진 최적화)

### 1.1 언어 설정
**현재 상태:**
- `_layouts/default.html`에서 `<html lang="en">`으로 설정되어 있음
- 블로그는 한국어 콘텐츠가 주를 이루고 있음

**개선 방안:**
- `<html lang="ko">`로 변경하거나 동적으로 설정
- 포스트별로 언어를 지정할 수 있도록 front matter에 `lang` 필드 추가

**영향도:** 높음 | **우선순위:** 높음

### 1.2 메타 태그 개선
**현재 상태:**
- 기본적인 Open Graph 태그는 구현되어 있음
- Twitter Card 메타 태그가 없음
- 포스트별 고유한 og:image가 없음 (현재는 sidebar-avatar만 사용)

**개선 방안:**
- Twitter Card 메타 태그 추가 (`twitter:card`, `twitter:title`, `twitter:description`, `twitter:image`)
- 포스트 front matter에 `og-image` 필드 추가하여 포스트별 썸네일 이미지 설정
- `og:image`의 기본값을 더 매력적인 이미지로 변경

**영향도:** 중간 | **우선순위:** 중간

### 1.3 구조화된 데이터 (Schema.org)
**현재 상태:**
- 구조화된 데이터가 전혀 없음

**개선 방안:**
- JSON-LD 형식으로 구조화된 데이터 추가:
  - `BlogPosting` 스키마 (포스트 페이지)
  - `Blog` 스키마 (메인 페이지)
  - `Person` 스키마 (About 페이지)
  - `BreadcrumbList` 스키마 (네비게이션)

**영향도:** 높음 | **우선순위:** 중간

### 1.4 sitemap.xml
**현재 상태:**
- `jekyll-sitemap` 플러그인 사용 중
- 설정 확인 필요

**개선 방안:**
- sitemap.xml이 제대로 생성되는지 확인
- robots.txt 파일 생성 및 검색 엔진 크롤링 최적화

**영향도:** 중간 | **우선순위:** 낮음

### 1.5 Canonical URL
**현재 상태:**
- `_includes/head.html`에 canonical URL이 구현되어 있음
- 하지만 favicon 링크에 오류가 있음 (line 58: `{{ site.baseurl }}https://...`)

**개선 방안:**
- favicon 링크 수정: `{{ site.baseurl }}` 제거 (이미 절대 URL)

**영향도:** 낮음 | **우선순위:** 낮음

## 2. 성능 최적화

### 2.1 이미지 최적화
**현재 상태:**
- 이미지는 WebP 형식으로 변환되어 있음 (좋음)
- CDN을 사용하고 있음 (jsDelivr)
- Lazy loading이 구현되어 있지 않음
- 이미지에 alt 속성이 없음

**개선 방안:**
- 모든 이미지에 `loading="lazy"` 속성 추가
- 포스트 내 이미지에 적절한 `alt` 텍스트 추가
- 반응형 이미지 (`srcset`, `sizes`) 구현 고려
- 이미지 최적화 자동화 (빌드 시 자동 압축)

**영향도:** 높음 | **우선순위:** 높음

### 2.2 CSS/JS 최적화
**현재 상태:**
- minified 버전 사용 중 (좋음)
- 하지만 모든 CSS/JS가 모든 페이지에 로드됨

**개선 방안:**
- 페이지별로 필요한 CSS/JS만 로드하도록 최적화
- Critical CSS 인라인화
- 사용하지 않는 CSS 제거 (예: Font Awesome이 주석 처리되어 있음)

**영향도:** 중간 | **우선순위:** 중간

### 2.3 폰트 최적화
**현재 상태:**
- Google Fonts (Noto Sans KR) 사용 중
- `preconnect`는 구현되어 있음

**개선 방안:**
- `font-display: swap` 추가
- 폰트 서브셋팅 고려 (한국어만 필요한 경우)
- 폰트 파일을 자체 호스팅하여 제3자 요청 감소

**영향도:** 중간 | **우선순위:** 낮음

### 2.4 서비스 워커 개선
**현재 상태:**
- Service Worker 구현되어 있음
- Stale-while-revalidate 전략 사용 중

**개선 방안:**
- 캐시 버전 관리 개선
- 오프라인 페이지 개선
- 백그라운드 동기화 고려

**영향도:** 낮음 | **우선순위:** 낮음

### 2.5 리소스 힌트
**현재 상태:**
- Google Fonts에 대한 `preconnect`만 있음

**개선 방안:**
- CDN 리소스에 대한 `dns-prefetch` 추가
- 중요 리소스에 대한 `preload` 추가
- 외부 스크립트에 대한 `prefetch` 고려

**영향도:** 낮음 | **우선순위:** 낮음

## 3. 접근성 (Accessibility)

### 3.1 이미지 alt 텍스트
**현재 상태:**
- 포스트 내 이미지에 alt 속성이 없음

**개선 방안:**
- 모든 이미지에 의미 있는 alt 텍스트 추가
- 장식용 이미지는 빈 alt (`alt=""`) 사용

**영향도:** 높음 | **우선순위:** 높음

### 3.2 키보드 네비게이션
**현재 상태:**
- 확인 필요

**개선 방안:**
- 모든 인터랙티브 요소가 키보드로 접근 가능한지 확인
- 포커스 인디케이터 명확히 표시
- 스킵 링크 추가

**영향도:** 중간 | **우선순위:** 중간

### 3.3 색상 대비
**현재 상태:**
- 확인 필요

**개선 방안:**
- WCAG 2.1 AA 기준에 맞는 색상 대비 확인
- 색상만으로 정보를 전달하지 않도록 주의

**영향도:** 중간 | **우선순위:** 중간

### 3.4 ARIA 레이블
**현재 상태:**
- ARIA 레이블이 부족함

**개선 방안:**
- 네비게이션, 검색, 버튼 등에 적절한 ARIA 레이블 추가
- 랜드마크 역할 명시

**영향도:** 중간 | **우선순위:** 낮음

### 3.5 스크린 리더 지원
**현재 상태:**
- 확인 필요

**개선 방안:**
- 스크린 리더로 테스트
- 의미론적 HTML 사용 확인
- 폼 요소에 적절한 레이블 연결

## 4. 보안

### 4.1 Content Security Policy (CSP)
**현재 상태:**
- CSP 헤더가 설정되어 있지 않음

**개선 방안:**
- CSP 헤더 추가하여 XSS 공격 방지
- 인라인 스크립트 최소화

**영향도:** 높음 | **우선순위:** 중간

### 4.2 외부 스크립트
**현재 상태:**
- Google Analytics, Google AdSense, Disqus 등 외부 스크립트 사용

**개선 방안:**

- 가능한 경우 `rel="noopener noreferrer"` 추가

**영향도:** 중간 | **우선순위:** 낮음

**현재 상태:**
- GitHub Pages는 기본적으로 HTTPS 제공

**개선 방안:**
- HSTS 헤더 확인
- Mixed Content 방지

**영향도:** 낮음 | **우선순위:** 낮음
